# Deterministic Cognitive Context → Provider Inference Injection Pattern

## When to use this pattern

You have a deterministic cognitive pipeline (V3 → V4 → V5 decision modules) that produces structured analysis, but the final answer must be a natural-language response generated by an LLM provider (DeepSeek, Claude, etc.). The deterministic output must reach the provider as context in the system prompt, not just be rendered solely by deterministic stripping.

## Architecture

```
User question
    │
    ├─→ Deterministic cognitive pipeline ──→ intent, matrix, confidence, recommended option
    │
    ├─→ Cognitive Context Pack (buildCognitiveContextPack)
    │     └─→ available_items.push(v5_decision_context)  ◄── INJECTION POINT
    │
    ├─→ Brain Context Builder (buildOpenClawBrainContext)
    │     └─→ formats available_items into system prompt context text
    │
    └─→ Provider POST (DeepSeek /chat/completions)
          └─→ systemContent = systemPrompt + contextText (includes V5 decision facts)
          └─→ Provider's answer uses V5 decision context naturally
```

## Implementation steps

### 1. Import the deterministic cognitive pipeline

In `buildBrainAskResponse` (or equivalent provider-backed handler):

```javascript
import { createV5RuntimeIntegration } from '../cognitive/cloe-v5-runtime-integration.js';
```

### 2. Run the deterministic pipeline after building the cognitive context pack

```javascript
let v5DecisionContext = null;
try {
  const v5Instance = createV5RuntimeIntegration();
  if (v5Instance && typeof v5Instance.isOperational === 'function'
      && v5Instance.isOperational()) {
    const decisionResult = v5Instance.enhanceWithV5
      ? v5Instance.enhanceWithV5({ ok: true }, promptText)
      : null;
    if (decisionResult && decisionResult.v5
        && decisionResult.v5.v5_pipeline_activated) {
      v5DecisionContext = decisionResult.v5;
    }
  }
} catch {
  v5DecisionContext = null;  // Non-blocking — existing behavior preserved
}
```

### 3. Inject into cognitive context pack's available_items

This makes the V5 context flow through `buildOpenClawBrainContext` → system prompt → provider:

```javascript
cognitiveContextPack.available_items.push({
  kind: 'v5_decision_context',
  name: 'V5DecisionCollector',
  description: `V5 decision context: intent: ${intent}; recommended: ${option}`,
  source: 'cloe_v5_decision_pipeline',
  available: true,
  read_only: true,
  safety_flags: ['read_only', 'no_network', 'no_write', 'no_execute'],
  metadata: {
    v5_pipeline_activated: true,
    decision_intent: { /* ... */ },
    recommended_option: '...',
    confidence_verdict: '...'
  }
});
cognitiveContextPack.needs_provider_context = true;
```

### 4. Add safe audit signals to the provider response

Prove the V5 context reached the provider without exposing secrets:

```javascript
v5_audit: v5DecisionContext ? {
  v5_provider_context_injected: true,
  v5_pipeline_activated: true,
  v5_modules_completed: v5DecisionContext.v5_modules_completed || 0,
  decision_classification: v5DecisionContext.decision_intent?.intent || null,
  recommended_option_available: Boolean(v5DecisionContext.decisive_answer?.recommended_option),
  confidence_available: Boolean(v5DecisionContext.decision_confidence?.verdict),
  provider: 'deepseek',
  model: 'deepseek-v4-flash'
} : {
  v5_provider_context_injected: false,
  v5_pipeline_activated: false,
  reason: 'V5 not triggered — non-decision prompt or modules unavailable'
}
```

## Verification

1. Check `v5_audit.v5_provider_context_injected === true` in the response
2. Check that the system content sent to the provider includes V5 decision facts
3. Verify all deterministic test suites still pass (no regression)
4. Run mock provider benchmark to validate end-to-end pipeline

## Pitfalls

- **createV5RuntimeIntegration returns the instance directly** — do NOT call it twice (`const v5 = createV5RuntimeIntegration()` is the instance, not a factory)
- **Cognitive context pack items array is mutable** — add items AFTER `buildCognitiveContextPack` returns, not before
- **Non-blocking failure** — wrap V5 pipeline in try/catch with empty handler so provider still works if V5 modules fail to load
- **Token bloat** — V5 decision context adds tokens to the system prompt; monitor context budget
